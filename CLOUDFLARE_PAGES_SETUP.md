# Cloudflare Pages Setup (Vite + React + TypeScript + Tailwind)

Use these exact values in Cloudflare Pages:

- Framework preset: `Vite`
- Build command: `npm run build`
- Build output directory: `dist`
- Root directory: `/` (leave blank if your repo root is this project)
- Node.js version: `20`

## Environment Variables (optional but recommended)

- `NODE_VERSION=20`

## Deploy Steps

1. Push this project to GitHub.
2. In Cloudflare Dashboard, open Pages.
3. Click `Create a project` and connect your GitHub repository.
4. Set the build values exactly as listed above.
5. Deploy.

## Notes

- Static assets must stay in `public/`.
- Your resume download is served from `/Maynard_Ermita_Resume.pdf`.
- Your profile photo is served from `/menong.jpg`.
- This project uses hash-section links (`#projects`, `#contact`) and does not require SPA rewrite rules.
- `index.html` is intentionally generated by `scripts/ensure-index.mjs` during `npm run dev`, `npm run build`, and `npm run preview`.

## Troubleshooting

If deploy logs show this line:

- Executing user deploy command: `npx wrangler deploy`

and then fail with `@cloudflare/vite-plugin` / Vite peer dependency errors, the project was created as a Worker build flow instead of a Pages build flow.

Fix:

1. Create a new Cloudflare Pages project (not Worker) from the same GitHub repo.
2. Use only:
	- Build command: `npm run build`
	- Build output directory: `dist`
3. Leave Deploy command empty.
4. Redeploy.
